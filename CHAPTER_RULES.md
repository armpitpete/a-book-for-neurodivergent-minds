# Chapter Rules

## Main rule

Each chapter must work as a separate reading unit.

A reader should be able to open any chapter and understand the useful point without reading the whole book first.

## Required pattern

Each chapter should include:

1. A short opening point
2. A plain explanation
3. A quick-use section
4. A few scripts or examples
5. A short recap

## Avoid

- Long dependency on earlier chapters
- Heavy theory before help
- Too many cross-references
- Repeating the same setup for too long
