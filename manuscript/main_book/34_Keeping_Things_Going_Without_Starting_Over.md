# Keeping Things Going Without Starting Over

## What this chapter is for

This chapter is about keeping useful things going without having to start over every time they slip.

A useful thing might be:

- a morning step
- a food plan
- a reminder
- a notebook
- a support request
- a weekly check
- a medication support
- an admin box
- a body-doubling arrangement
- a rest plan
- a boundary
- a way of asking for help

Useful things often slip.

They get interrupted by illness, overload, tiredness, work, school, caring, money problems, appointments, emotions, or unexpected change.

When something slips, you may feel that all progress has gone.

You may think:

> I ruined it.

Or:

> I have to start again from zero.

Or:

> There is no point because I never keep things going.

This chapter has one main point:

> a useful thing does not have to work perfectly to still be worth keeping.

The important skill is not never dropping it.

The important skill is knowing how to return.

Chapter 33 was about getting help without being taken over.

Chapter 34 is about keeping useful supports and changes alive over time, even when they slip.

## If you can only read one thing

Do not ask first:

> How do I make sure I never slip again?

Ask:

> Where can I return when this slips?

Then ask:

> What would make returning easier?

You do not need a perfect streak.

You need a return point.

## Start here if

Start here if any of these are true:

- you stop using a helpful thing as soon as you miss a day
- you feel that a broken streak means failure
- you restart systems from the beginning again and again
- you abandon useful tools because you used them imperfectly
- you need a way back after illness, overload, shutdown, travel, school, work, or appointments
- you want a support or routine that can survive real life
- you need maintenance, not another complete redesign

## Short version

Useful supports can slip.

Slipping is not the same as losing all progress.

Useful first steps:

1. name what helped
2. name where it usually slips
3. make a return point
4. make the return point visible
5. adjust the support if it keeps breaking in the same place

Starting over is not always needed.

Sometimes you only need to come back at the smallest useful point.

## Slipping is not the same as losing everything

Many people treat a missed day as proof that the whole system has failed.

Examples:

- one missed medication support becomes “I am useless”
- one missed journal entry becomes “I cannot keep a notebook”
- one messy week becomes “planning never works”
- one unanswered message becomes “I am bad at people”
- one broken routine becomes “I need a completely new routine”

A slip means something interrupted the pattern.

It does not always mean the whole thing was wrong.

Ask:

> Did this stop helping, or did I need an easier way back?

That question is kinder and more useful.

## Return points

A return point is the smallest place where you can rejoin a useful thing.

It is not the full version.

It is the way back in.

Examples:

- open the notebook, not write a full page
- put the letter in the admin box, not clear all admin
- take the next dose as prescribed, not analyse the whole medication history
- put one glass of water where you sit, not rebuild the whole food plan
- send one line, not answer every message
- do the first step of the morning routine, not the whole routine perfectly

A return point should be easy to recognise.

It should answer:

> What do I do when I come back?

## Make the return point visible

Return points work better when they are visible.

Examples:

- a note on the notebook: “Open here.”
- an admin box with one label: “Letters here.”
- a checklist with the first step circled
- medication support placed where it is safely visible
- a phone note called “When I fall behind”
- a message template ready to copy
- a bag kept in the same place

The visible return point should not shout at you.

It should guide you back.

## Maintenance, adjustment, and replacement

When something slips, there are three possible responses.

### Maintenance

Maintenance means keeping the same support and making it easier to use again.

Examples:

- put the notebook back where you see it
- refill the pill organiser if you use one
- clear the admin box enough to use it again
- put the checklist back on the door
- set the reminder back to the useful time

Use maintenance when the thing mostly works but got interrupted.

### Adjustment

Adjustment means changing the support so it fits better.

Examples:

- make the checklist shorter
- change the reminder time
- move the admin box closer
- reduce the weekly check to three questions
- ask for one reminder instead of repeated reminders
- change a morning routine from five steps to two steps

Use adjustment when the thing helps but keeps breaking in the same place.

### Replacement

Replacement means letting go of a support that does not fit and using a different route.

Examples:

- use voice notes instead of a written journal
- use a box instead of folders
- use written instructions instead of phone explanations
- use body-doubling instead of trying alone
- use one-page notes instead of a full planner

Replacement is not failure.

Sometimes the old support was asking your brain to work in the wrong way.

## When a system does not fit

Not every support needs saving.

Some systems fail because they are wrong for you.

A system may not fit if:

- it needs too many steps
- it depends on memory you do not reliably have
- it only works on high-capacity days
- it makes you feel watched or punished
- it creates more admin than it removes
- it relies on an unsafe person
- it turns every slip into shame

If a system only works when you are already doing well, it may not be the right system for hard days.

You are allowed to change the system.

You are allowed to choose a smaller, less impressive, more usable support.

## What helps

### 1. Keep the useful part

When something breaks, do not throw away the useful part too quickly.

Ask:

> What part of this actually helped?

Examples:

- the notebook helped because it held thoughts
- the reminder helped because it made time visible
- the support person helped because they sat quietly
- the admin box helped because letters had one place
- the checklist helped because it reduced decisions

Keep the useful part if you can.

Change the part that made it too hard.

### 2. Make the restart smaller

A full restart can feel too heavy.

A small return is easier.

Examples:

- not “restart the whole planner”
- use “write today’s date”

- not “catch up with all messages”
- use “answer one important message”

- not “fix the whole room”
- use “clear one safe surface”

Small returns protect movement.

### 3. Build for bad days

A support that only works on good days may not be enough.

Ask:

> What is the bad-day version?

Examples:

- good day: cook a planned meal
- bad day: eat safe simple food

- good day: write a full note
- bad day: write one word or take one photo

- good day: clear all admin
- bad day: put letters in the box

- good day: full morning routine
- bad day: medication, food, clothes, leave

The bad-day version is not a lower moral standard.

It is a survival design.

### 4. Leave a breadcrumb for later

A breadcrumb is a small clue that helps future-you return.

Examples:

- “Start here tomorrow.”
- “Next step: call pharmacy.”
- “Letter opened; deadline is Friday.”
- “Bag needs keys.”
- “This form needs page 3 next.”

Breadcrumbs reduce the cost of returning.

They help when memory, energy, or words are low.

### 5. Stop rebuilding from scratch

Starting from scratch can become its own trap.

It can feel exciting for a short time.

But it can also use all your capacity before the support begins working.

Before rebuilding, ask:

- What already helped?
- What broke?
- What can be made smaller?
- What return point is needed?
- What can stay the same?

You may not need a new system.

You may need an easier way back into the old useful part.

## What to try now

### 30 seconds

Finish this sentence:

> One useful thing I want to return to is ____.

Examples:

- admin box
- reminder
- notebook
- rest plan
- food support
- morning step
- body-doubling
- message template

Name one thing.

That is enough.

### 3 minutes

Write three lines:

- Useful thing:
- Where it slips:
- Smallest return point:

Example:

- Useful thing: admin box
- Where it slips: when letters pile up
- Smallest return point: put all letters in the box without sorting them

### 30 minutes

Make a return point map.

Use these headings:

1. What helped before?
2. Where does it usually slip?
3. What is the smallest useful return point?
4. How can I make the return point visible?
5. Does this need maintenance, adjustment, replacement, or a bad-day version?

Keep it short.

This is not a full redesign.

It is a way back in.

## Useful scripts

### Returning after a gap

“I do not have to start from zero. I can return at the smallest useful point.”

### Keeping the useful part

“This part helped. I can keep it and change the part that broke.”

### Asking for support to return

“Can you help me find the next step without rebuilding the whole system?”

### Making the version smaller

“I need the bad-day version, not the perfect version.”

### Avoiding a full restart

“I am not making a new system today. I am making this one easier to return to.”

### Letting a system go

“This does not fit me well enough. I can replace it without calling myself a failure.”

## What not to force

Do not treat every missed day as failure.

Do not rebuild the whole system every time something slips.

Do not keep a system that only works on high-capacity days.

Do not use streaks to punish yourself.

Do not make the return point so large that you avoid it.

Do not throw away the useful part before checking whether it can be adjusted.

Do not keep a system that makes you less safe, more ashamed, or more stuck.

## You are not failing if

You are not failing if:

- a useful thing slips
- you need a smaller return point
- you need the bad-day version
- you need reminders to come back
- you need support to restart
- you replace a system that does not fit
- you keep returning instead of staying perfectly consistent

Progress is not only perfect continuation.

Progress can also be coming back sooner, with less shame and less damage.

## Quick summary

- Useful things often slip; slipping does not always mean losing all progress.
- A return point is the smallest place where you can rejoin a useful support.
- Keep the useful part, adjust what breaks, and stop starting from zero every time.
