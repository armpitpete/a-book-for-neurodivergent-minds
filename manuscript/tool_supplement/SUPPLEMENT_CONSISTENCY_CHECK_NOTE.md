# Supplement Consistency Check Note

## Status

Draft v0.1.

Created after the 12-tool supplement structure was completed.

## Purpose

This note checks consistency across the supplement tools.

It is not a final proofread.

It identifies what is already working and what should be tightened before the supplement is treated as final.

## Overall result

The supplement is structurally coherent.

The 12 tools now work as a practical support set rather than a second book.

The order is usable:

1. immediate pressure
2. good-enough stopping line
3. next safe action
4. capacity
5. body floor
6. choices
7. support ask
8. noise/load
9. transitions
10. boundaries/exits
11. misreads
12. handover

This gives the reader a path from immediate stabilising action toward communication, repair, and continuation.

## Format consistency

The tools now broadly share the same structure:

- status
- what this tool is for
- use this when
- default line
- 30 seconds
- 3 minutes
- 30 minutes
- script/template
- stop point
- linked chapters

This is good.

It means the reader learns the pattern once and can then use any tool without a new learning demand.

## Strong repeated design choices

These repeated choices are useful and should stay:

- one default line per tool
- 30-second version first
- practical examples
- explicit stop point
- shame-reducing language
- “one action counts” logic
- scripts that can be copied directly
- permission to stop before perfection

## Repetition to watch

Some repetition is intentional because the supplement is for bad-day use.

However, a later tightening pass should watch for overuse of these patterns:

- “The aim is not...”
- “Do not turn this into...”
- “The tool has worked if...”
- repeated “Use this when” phrasing
- repeated “one ___ counts” phrasing

These are not errors.

They are useful scaffolding.

But if every tool sounds too identical, the reader may feel the book is repeating itself mechanically.

## Tone check

The tone is mostly right for the project:

- plain
- direct
- practical
- low-shame
- young-person-readable
- adult-respectful
- not clinical-heavy
- not motivational in a forced way

The tools avoid making the reader prove distress before using support.

The tools also avoid turning neurodivergent difficulty into moral failure.

## Length check

Most tools are usable, but several may be slightly long for a final supplement page.

Likely longer tools:

- Tool 5 — The Body Floor Check
- Tool 9 — The Transition Bridge
- Tool 10 — The Boundary/Exit Card
- Tool 11 — The Misread Map
- Tool 12 — The Quiet Handover

These are not too long for draft stage.

For final layout, consider giving each tool:

- one main page
- one optional extended script section
- one printable card version

## Reader-facing cleanup needed

Most modules still include project-facing status lines such as:

- “Draft v0.1”
- “Created from the current 12-tool supplement sequence”
- “Updated to match the current 12-tool supplement format”

These are useful while drafting.

They should probably be removed from final reader-facing supplement pages, or moved into frontmatter.

Suggested later frontmatter pattern:

```yaml
---
tool: 1
title: The One-Pressure Check
status: draft_v0.1
linked_chapters: [1, 3, 4]
---
```

Then the reader-facing page can begin with the tool title and the practical text.

## Linked chapter check needed

The linked chapter sections are useful, but they need a later title check against final chapter names.

Possible risks:

- some linked chapter titles may not exactly match final filenames
- some chapters have changed role language during drafting
- some tools may link to too many chapters for a short supplement

Good enough for now: the links are conceptually useful.

Final pass: confirm exact chapter titles and reduce links if the list feels heavy.

## Tool-by-tool check

### Tool 1 — The One-Pressure Check

Works as the opening tool.

It is immediate, practical, and low-demand.

Final pass: consider whether Tool 1 and Tool 8 overlap too much. Tool 1 reduces one pressure now; Tool 8 budgets total load. Keep that distinction clear.

### Tool 2 — The Good-Enough Line

Strong role.

It protects the reader from turning the book or a task into another demand.

Final pass: keep this near the front of the supplement.

### Tool 3 — The Next Safe Step

Strong role.

It converts the older “First Stupid Step” idea into safer language.

Final pass: keep “safe” and “real” as the main idea, not productivity.

### Tool 4 — The Capacity Check

Strong role.

It supports uneven skills and low-capacity days.

Final pass: check that it does not sound like a scoring system.

### Tool 5 — The Body Floor Check

Strong and necessary.

It may be one of the most practically useful tools.

Final pass: check health wording carefully so it does not sound like body basics replace medical help.

### Tool 6 — The Choice Shrinker

Strong role.

It gives a clear way to reduce decision paralysis.

Final pass: keep “safe-enough” wording consistent.

### Tool 7 — The Support Ask

Strong role.

It gives practical support language without requiring performance of distress.

Final pass: keep the non-speaking note version.

### Tool 8 — Noise Budget

Strong original concept.

The shelf/load idea remains useful and memorable.

Final pass: make sure it does not duplicate Tool 1 too closely.

### Tool 9 — The Transition Bridge

Strong role.

It covers starting, stopping, leaving, returning, and switching.

Final pass: check length and consider a shorter printable version.

### Tool 10 — The Boundary/Exit Card

Strong role.

It successfully includes the older Exit Early and Boring + Bounded ideas.

Final pass: check that examples stay safe, plain, and non-escalating.

### Tool 11 — The Misread Map

Strong role.

Intent / impact / assumption wording is clear and useful.

Final pass: make sure it does not over-push repair when a boundary or no further action is safer.

### Tool 12 — The Quiet Handover

Strong closing tool.

It supports shutdown, appointments, next-day continuation, and low-speech moments.

Final pass: keep this as the final tool because it closes the supplement by helping the reader continue without restarting.
